# SKILL.md 작성 포맷 가이드

## 기본 구조

~~~
---
name: 스킬명
description: 한 줄 설명.
  트리거 문구 1, 트리거 문구 2 같은 요청에 사용한다.
---

## 절차 (or 모드 판단)
1. 단계
2. 단계

## 행동 규칙
- 규칙 (긍정 + 부정 지시 모두 포함)

## 출력 형식 (필요시)
### 섹션명
(설명)
~~~

---

## frontmatter 규칙
- name: 스킬 디렉토리명과 동일하게
- description: 200자 이내, 트리거 문구 반드시 포함

나쁜 예: "코드 관련 스킬"
좋은 예: "코드를 깊이 분석하고 문제점을 제안한다.
  '코드 리뷰', '이 코드 봐줘' 같은 요청에 사용한다."

---

## 패턴 선택 기준
- Pattern A: Claude 판단력만으로 충분 → SKILL.md만 (대부분의 경우)
- Pattern B: 결정론적 처리 필요 (계산, 파일 변환) → scripts/ 추가
- Pattern C: 외부 서비스 연동 필요 → MCP/서브에이전트

대부분 Pattern A로 시작하고 필요시 확장

---

## 작성 규칙
- 50줄 이하로 간결하게
- Claude가 이미 아는 내용 넣지 않기
- 부정 지시 반드시 포함 ("~하지 말 것")
- 직접 수정은 "수정해줘" 요청 시에만 실행
- 예시는 Input/Output 쌍으로

---

## 예시 (conventional-commit)

~~~
---
name: conventional-commit
description: git 커밋을 수행하거나 커밋 메시지를 작성한다.
  "커밋해줘", "commit", "변경사항 커밋" 같은 요청에 사용한다.
---

## 절차
1. `git diff --staged`로 스테이징된 변경사항 확인
2. 변경 내용 분석 후 type 선택
3. 커밋 메시지 작성 후 실행

## 행동 규칙
- 스테이징된 파일 없으면 git status 확인 후 안내
- 이모지 사용 금지
~~~
