## 절차

### 1. 스테이징 상태 확인
`git status`로 스테이징된 파일 여부 확인

### 2-A. 스테이징된 파일이 있는 경우
1. `git diff --staged`로 스테이징된 변경사항 확인
2. 변경 내용 핵심 요점으로 요약
3. 위험 요소 체크

### 2-B. 스테이징된 파일이 없는 경우
1. `git diff HEAD`로 전체 변경사항 확인
2. 변경된 파일 목록과 각 파일별 변경 내용 요약
3. 관련된 파일끼리 묶어서 커밋 단위로 스테이징 제안
4. 위험 요소 체크

## 위험 요소 체크리스트

### 보안
- 하드코딩된 값 (API 키, URL, 매직 넘버)
- .env나 시크릿 파일 포함 여부

### 안정성
- 에러 핸들링 누락
- 기존 인터페이스 변경 (breaking change)
- 테스트가 필요한데 빠진 부분

### 코드 품질
- 디버그용 코드 잔존 (print, console.log, breakpoint 등)
- 불필요한 코드 (dead code, 주석 처리된 코드)
- 중복 코드 (동일하거나 유사한 로직이 여러 곳에 존재)
- TODO/FIXME 주석 잔존

## 출력 형식 (2-A: 스테이징 있음)
### 변경 요약
(핵심 변경사항 2~3줄)

### 위험 요소
(있으면 카테고리별 목록, 없으면 "없음")

## 출력 형식 (2-B: 스테이징 없음)
### 변경된 파일 목록
(파일명 + 한 줄 요약)

### 커밋 단위 제안
(관련 파일끼리 묶어서 git add 명령어 포함)

### 위험 요소
(있으면 카테고리별 목록, 없으면 "없음")
