## 모드 판단
- 현재 디렉토리에 코드/파일이 있으면 → 기존 프로젝트 모드
- 빈 디렉토리이거나 코드가 없으면 → 새 프로젝트 모드

## 기존 프로젝트 모드 절차
1. 프로젝트 구조 탐색
   - 디렉토리 구조, 기술 스택 파악
     (pyproject.toml, package.json, requirements.txt 등)
   - 핵심 파일/모듈 파악
2. references/claude_md_guide.md 읽고 작성 기준 숙지
3. 작성 규칙에 따라 CLAUDE.md 초안 작성
4. 프로젝트 루트에 CLAUDE.md 저장

## 새 프로젝트 모드 절차
1. 다음 항목을 사용자에게 질문
   - 어떤 프로젝트인지 (한 줄 설명)
   - 기술 스택
   - 주요 설계 결정이나 규칙 (있으면)
2. references/claude_md_guide.md 읽고 작성 기준 숙지
3. 답변 기반으로 CLAUDE.md 초안 작성
4. 프로젝트 루트에 CLAUDE.md 저장

## 작성 규칙
- 200줄 이하, 가능하면 50~100줄
- Claude가 파일을 읽어서 알 수 있는 정보는 넣지 않기
  (기술 스택 상세, 의존성 목록, DB 스키마 전체 등)
- 설계 의도와 규칙 위주로 작성
- TODO, 로드맵 내용 포함 금지
- 상세 내용은 @파일경로로 참조

## 포함할 섹션
- 프로젝트: 한두 문장 요약
- 기술 스택: 프레임워크 + DB + 핵심 라이브러리 한 줄
- 핵심 경로: 자주 건드리는 파일/디렉토리 5~10개 (기존 프로젝트만)
- 설계 규칙: Claude가 어기면 버그 나는 것들
- 현재 상태/약점: 알려진 문제점 (있을 때만)
- 참조 문서: 상세 내용은 경로만

## 금지 사항
- 환경변수 목록 포함 금지 (.env에서 읽으면 됨)
- 서버 실행 방법 포함 금지
- 코드 예시 포함 금지 (Skills로 분리)
- 200줄 초과 금지
