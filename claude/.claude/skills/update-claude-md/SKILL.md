## 모드 판단
- 실수/규칙 추가 ("~하는 실수 했어", "~규칙 추가해줘") → 시나리오 A
- 프로젝트 변화 ("~로 바꿨어", "~제거했어", "~추가됐어") → 시나리오 B
- 정리 요청 ("정리해줘", "다듬어줘") → 시나리오 C

## 시나리오 A: 실수 기록 → 규칙 추가
1. 현재 CLAUDE.md 읽기
2. 실수 내용을 규칙 형태로 변환
   - 나쁜 예: "date 컬럼은 ISO 형식이고 이건 시계열 분석을 위해서이며..."
   - 좋은 예: "date 컬럼: ISO 형식만 사용. date_display: 한국어 표시용. 혼용 금지."
3. 가장 적합한 섹션에 한 줄 추가
4. 변경 내용 보여주고 확인 후 저장

## 시나리오 B: 프로젝트 변화 → 내용 수정
1. 현재 CLAUDE.md 읽기
2. 변경된 내용과 관련된 기존 항목 파악
3. 수정/삭제/추가할 내용 정리해서 보여주기
4. 확인 후 저장

## 시나리오 C: 정기 정리
1. 현재 CLAUDE.md 읽기
2. 아래 기준으로 점검
   - 더 이상 해당 안 되는 규칙
   - 중복되거나 합칠 수 있는 규칙
   - Skills/docs로 분리할 수 있는 내용
   - 설명이 아닌 규칙 형태로 바꿀 수 있는 항목
3. 정리 제안 내용 보여주고 확인 후 저장

## 줄 수 경고
- 200줄 이하: 정상
- 200~300줄: 저장 후 경고
  "200줄을 초과했습니다. Skills/docs로 분리할 내용이 있는지 확인해보세요."
- 300줄 초과: 저장 후 강력 경고
  "300줄을 초과했습니다. 분리를 강력히 권장합니다."
  + 분리 후보 항목 제안

## 공통 규칙
- 수정 전 반드시 변경 내용 보여주고 확인 후 저장
- CLAUDE.md가 없으면 "/create-claude-md를 먼저 실행하세요" 안내
- 설명 문장은 규칙 형태로 변환 (짧고 명확하게)
