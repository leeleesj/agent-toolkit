name: code-review
description: 코드를 깊이 분석하고 문제점과 개선점을 제안한다.
  "코드 리뷰", "이 코드 봐줘", "리뷰해줘", "코드 품질 확인" 같은
  요청에 사용한다. 파일을 지정하면 해당 파일을, 없으면 변경된 코드를 리뷰한다.
---

## 절차
1. 대상 파일 또는 `git diff HEAD`로 변경된 코드 확인
2. 필요시 관련 파일 추가 확인
3. 로직/보안/성능/유지보수성/코드품질 순으로 점검
4. 심각도별로 정리해서 출력
5. 직접 수정은 "수정해줘" 요청 시에만 실행

## 행동 규칙
- 각 이슈에 심각도 표시: critical / warning / suggestion
- 파일명과 라인번호 반드시 명시
- 문제점 + 이유 + 수정 코드 예시를 함께 제시
- 이슈 없으면 "이슈 없음"으로 간결하게 종료

## 출력 형식
### 리뷰 요약
(전반적인 코드 품질 한 줄 평가)

### 발견된 이슈
[critical/warning/suggestion] 파일명:라인번호
문제: (무엇이 문제인지)
이유: (왜 문제인지)
수정 제안:
```코드
(수정 예시)
```

### 총평
- critical: N개 / warning: N개 / suggestion: N개
